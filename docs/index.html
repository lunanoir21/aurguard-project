# Why

AUR helpers happily run untrusted build code as your user — and .install scriptlets as root. A typo-squatted package, a hijacked maintainer account, or a single curl … | sh line is all it takes. aurguard puts a review step in front of that.

🧬

Clone-first, TOCTOU-safe

The repo is cloned once, then the same tree is analyzed and built — no swap between what you read and what runs.

🌳

Real bash parsing

A tree-sitter-bash AST sees eval and pipe-into-shell through quoting tricks, comments, and here-docs.

🚫

Never executes

Analysis is purely static. A clean report means “no known-bad shapes,” and nothing in the package has run.

# What it checks

Twelve analysis passes feed one report: a tree-sitter-bash AST pass, textual heuristics, a 24-rule YARA-style signature database (plus an updatable rules.d/ overlay), recursive decoding, anti-evasion normalization, dataflow taint, IOC/wallet matching, committed-binary + VirusTotal checks, a source-tree text scan over every build-helper script, PGP/pkgver() integrity, and package metadata & history.

🔎

24 YARA-style signatures

A dependency-free rule engine matches named malware shapes — miners, exfiltration, persistence, defense-evasion, container escape, cloud-metadata SSRF, reverse shells — with vetoes that stay quiet on legitimate $pkgdir packaging. Extend it with your own rules in config.

🧬

Anti-evasion decoding

Recursively decodes nested base64/hex, folds b""ash / ${IFS} / $'\x65' tricks, reverses rot13 / reversed / URL-encoded strings, and flags base64-wrapped gzip/ELF payloads — then re-runs every signature over what it finds.

🩸

Dataflow taint

Connects untrusted input to execution across lines: a captured $(curl …) or a downloaded file reaching eval, sh -c, a here-string, or source — the split-fetch-then-run trick that single-line rules miss.

🧪

Committed binaries & VirusTotal

Prebuilt executables checked into a package tree are flagged and SHA-256-hashed. Check them on VirusTotal — offline by hash + link, or via the API with --vt. Nothing is ever uploaded.

🌐

Source & metadata

Insecure/IP/shortened source URLs, SKIP checksums, untrusted git clone, maintainer age, low votes, staleness, and pkgrel churn.

🔐

Root scriptlets

.install hooks run as root, so they get the full rule set plus a flag for any network access at install time.

📜

Source-tree text scan

Malware doesn't have to live in the PKGBUILD. Every *.sh, *.py, *.js, Makefile, build.rs, and unreferenced .install file in the cloned tree gets the same AST + signature + decode + taint treatment, attributed by path:line.

🔄

Updatable ruleset

aurguard --update-rules fetches a versioned TOML signature set over HTTPS, validates it before touching disk, and never downgrades. Layer your own rules in rules.d/ without recompiling.

📝

Approval ledger

Approve a PKGBUILD once; aurguard surfaces silent upstream edits on the next install via a content digest.

⏿

Scriptable

--json output and exit codes (0/10/20) make it a clean CI gate.

🗣️

Five languages

Interface and findings localized end to end.

# A sample of the rules

Findings are bucketed INFO · WARN · CRITICAL; a package's risk is the highest severity it trips. CRITICAL is blocked by default.

CURL_PIPE_SH REVERSE_SHELL CRYPTO_MINER DISCORD_EXFIL CRED_HARVEST SSH_KEY_INJECT CRON_PERSIST DECODED_THREAT COMPRESSED_PAYLOAD TAINTED_EXEC PKGVER_EXEC COMMITTED_BINARY IOC_MATCH VT_FLAGGED LD_PRELOAD_HIJACK CLOUD_METADATA DOCKER_SOCK SUDOERS_TAMPER PY_REVERSE_SHELL DESTRUCTIVE EVASION_NORMALIZED WALLET_ADDRESS MISSING_PGP MAINTAINER_CHANGED DELTA_NEW_RISK ORPHAN_PACKAGE FLAGGED_OUTDATED UNKNOWN_SOURCE TOR_C2 PIP_INDEX_HIJACK WEBHOOK_EXFIL_GENERIC CURL_BASH_PROCSUB CUSTOM_RULE

…and more — 24 signature rules plus the heuristic, decode, taint, metadata, and integrity passes. Add your own rules and IOC lists in config.toml.

# Install

Linux only — the AUR is Arch-specific.

From source (recommended) 

# needs the Rust toolchain + a C compiler (tree-sitter-bash grammar)
git clone https://github.com/lunanoir21/aurguard-project.git
cd aurguard-project
cargo build --release
./target/release/aurguard --setup

From npm 

npm install -g aurguard

The npm package is a thin launcher with no install script. The prebuilt binary ships as a platform-specific optional dependency, so nothing is downloaded or executed at install time — no postinstall, no remote-code surface, zero runtime dependencies.

From crates.io 

cargo install aurguard
~ aurguard
